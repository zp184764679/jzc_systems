# 🔧 端口变更通知

## 问题已解决！

PM2疯狂重启的问题已经修复。原因是**端口冲突** - 8001-8006端口都被其他系统占用了。

## 新的端口配置

### 报价系统后端
- **旧端口**: 8001
- **新端口**: **9001** ✅
- **健康检查**: http://localhost:9001/health

### 前端配置
- 已自动更新 `.env` 文件指向新端口
- 前端会自动连接到 `http://localhost:9001`

## 变更详情

1. **后端启动脚本** (`backend/start.py`): 默认端口改为 9001
2. **前端环境配置** (`frontend/.env`): API地址改为 `http://localhost:9001`
3. **PM2配置**: 已保存，重启后自动使用新端口

## 如何验证

```bash
# 测试后端健康状态
curl http://localhost:9001/health

# 检查PM2状态
pm2 list

# 查看后端是否稳定运行（不再频繁重启）
pm2 logs quotation-backend --lines 20
```

## 为什么选择9001端口？

经检查发现：
- 8001-8006 已被其他系统后端占用
- 9001端口空闲且未冲突
- 避免与现有服务冲突

## 下次重启

PM2配置已保存，系统重启后会自动使用9001端口。无需手动修改。
